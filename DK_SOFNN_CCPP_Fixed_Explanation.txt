WHAT IS THIS NOTEBOOK?
-----------------------
This notebook implements and evaluates the DK-SOFNN (Data-Knowledge-Driven
Self-Organizing Fuzzy Neural Network) algorithm on the Combined Cycle Power
Plant (CCPP) dataset. The goal is to predict the Electrical Output (PE, in MW)
of a power plant from four sensor readings, under a "domain shift" scenario
where the source and target domains have different label distributions.

It also trains and compares three baseline methods (GM-FTL, RS-SOFNN, APSO-FNN)
to prove that DK-SOFNN is superior.

CELL 2 — Data Loading & Preprocessing
--------------------------------------------------------------------------------
PURPOSE : Load the CCPP Excel file and split/normalise the data into source
          and target domains, simulating domain shift.

DATASET : Folds5x2_pp.xlsx
  - 9,568 samples, 5 columns: AT (temperature), V (vacuum), AP (pressure),
    RH (humidity), PE (electrical output).

SPLITTING STRATEGY (Section IV-A-2 of the paper):
  - Source domain    : first 1,500 samples → used to train source FNN
  - Target train     : next 100 samples   → used to train target FNN
                       (with artificial noise added to outputs)
  - Target test      : next 300 samples   → used to evaluate all models
                       (uses TRUE clean labels — no noise)

DOMAIN SHIFT SIMULATION:
  - The paper adds Gaussian noise N(mu=50, sigma=10) in MW to target training
    labels. This simulates a real-world scenario where the two domains have
    different output distributions (e.g., different operating conditions).
  - WHY? This forces the model to learn under "corrupted" supervision, testing
    robustness to label noise — a key challenge DK-SOFNN is designed to handle.

NORMALISATION:
  - Features (X) : normalised to [0,1] using min-max from SOURCE domain.
    Formula: X_norm = (X - X_min) / (X_max - X_min)
  - Outputs (y)  : normalised using source output range (ymin, ymax).
    Formula: y_norm = (y - y_min) / (y_max - y_min)
  - Both target train AND test use the source normalisation constants so that
    all domains share the same numerical scale.

KEY FIX (vs the original buggy version):
  - Test labels use TRUE values (no noise) so that RMSE measures real accuracy.
  - The noisy labels are only used for training (as in the paper).

--------------------------------------------------------------------------------
CELL 3 — RBFFNN Class Definition
--------------------------------------------------------------------------------
PURPOSE : Define the core neural network model — a Radial Basis Function
          Fuzzy Neural Network (RBFFNN).

ARCHITECTURE:
  Input layer (P features) → RBF hidden layer (K fuzzy rules) → Output (scalar)

PARAMETERS (per rule k):
  - C[k]  : center vector (P-dimensional) — "where" the rule fires in input space
  - S[k]  : width vector  (P-dimensional) — "how broadly" the rule fires
  - W[k]  : output weight (scalar)        — "how much" this rule contributes

KEY METHODS:
  forward(x)   : Computes the network output for a single input x.
  update(x,y,yd,...) : Performs one gradient-descent parameter update.
  add_rule(x, l)    : Grows the network by adding a new rule.
  del_rule(l)       : Prunes the network by removing rule l.
  replace_rule(...) : Replaces a target rule with a source rule (compensation).
  predict(X)        : Runs forward() for all samples in matrix X.

See the Formulas Reference file for the exact equations used here.

--------------------------------------------------------------------------------
CELL 4 — Fuzzy Rule Index Functions
--------------------------------------------------------------------------------
PURPOSE : Compute three health indices for each fuzzy rule, which are used to
          decide whether to grow, prune, or keep the network structure.

THE THREE INDICES (Eqs. 10-12):
  - R (Similarity)   : How "redundant" is rule l? High R = redundant (bad).
  - M (Sensitivity)  : How "active" is rule l? High M = important (good).
  - C (Contribution) : How "useful" is rule l for output? High C = important (good).

WINDOW-BASED COMPUTATION:
  - Indices are computed over a sliding window of the last N_win data points.
  - This keeps computation efficient and allows the structure to adapt online.

See the Formulas Reference file for exact computations.

--------------------------------------------------------------------------------
CELL 5 — Source FNN Training
--------------------------------------------------------------------------------
PURPOSE : Train the RBFFNN on the large source domain (1,500 samples) where
          labels are clean. This produces a well-trained "teacher" network.

ALGORITHM (Self-Organizing, Eqs. 8-15):
  For each epoch:
    For each sample (x, y_true):
      1. Forward pass → predicted y
      2. Gradient update of parameters (W, C, S) using Eq. 8-9
      3. Every N_win steps: compute R, M, C indices and decide:
         - GROW   if one rule has: min-R AND max-M AND max-C  (Eq. 13)
         - PRUNE  if one rule has: max-R AND min-M AND min-C  (Eq. 15)

WHY SELF-ORGANIZING?
  Starting with K0 rules (e.g., 10 or 20), the network adjusts its complexity
  automatically. It grows when it finds a rule that handles a distinct input
  region, and prunes rules that are redundant or inactive.

RESULT: A source FNN with K rules (typically converges to 6-20 rules) and
        low RMSE on source domain data.

Two source FNNs are trained (K0=10 and K0=20) to show structure stability
in Fig 11 of the paper.

--------------------------------------------------------------------------------
CELL 6 — DK-SOFNN Target Domain Training
--------------------------------------------------------------------------------
PURPOSE : Train the target domain FNN using BOTH data from the target domain
          AND knowledge from the source FNN.

This is the MAIN CONTRIBUTION of the DK-SOFNN paper. It uses TWO mechanisms:

  MECHANISM 1: Structure Compensation Strategy (Eqs. 17-22)
  ----------------------------------------------------------
  The source FNN's structure (number and arrangement of rules) serves as a
  "reference" for how the target FNN should be structured.

  Reference thresholds (computed on source domain):
    Rbar_s = mean of R-indices from source FNN predictions on source data
    Mbar_s = mean of M-indices from source FNN predictions on source data
    Cbar_s = mean of C-indices from source FNN predictions on source data

  During target training (every N_win steps), current target indices are
  computed and compared to these thresholds:

  GROW   (Eq. 17): Rbar_t <= Rbar_s  AND  Mbar_t >= Mbar_s  AND  Cbar_t <= Cbar_s
    → Target network is less redundant and more active than source → add a rule
    → Add rule centered at current input, based on highest-C rule

  PRUNE  (Eq. 19): Various conditions based on Rbar_t >= Rbar_s (too redundant)
    → Target network is more redundant than source → remove a rule
    → Several sub-conditions select WHICH rule to remove

  COMPENSATE (Eq. 20): When structure is degraded but full prune not triggered
    → Replace the weakest target rule with the strongest source rule
    → Uses: replace_rule(worst_target_rule, best_source_rule)

  MECHANISM 2: Parameter Reinforcement (Eqs. 24-30)
  ---------------------------------------------------
  When updating parameters, a knowledge regularisation term pulls the target
  parameters toward source parameters.

  For each sample, H different (alpha, beta) combinations are tried:
    alpha controls data weight: alpha*error_gradient
    beta  controls source pull: beta*(target_param - source_param)
    (alpha + beta = 1, with alpha in [0.5, 1.0])

  The combination that gives lowest NEXT-STEP error (lookahead, Eq. 28)
  is selected as the final parameter update (Eq. 30).

INITIALISATION:
  The target FNN starts as a CLONE of the source FNN (same C, S, W).
  Then adapts via the two mechanisms above.

--------------------------------------------------------------------------------
CELL 7 — Comparison Methods
--------------------------------------------------------------------------------
PURPOSE : Define three baseline methods for fair comparison.

  GM-FTL (General Model for Fine-TuneaLearning):
    - Transfers source structure to target (clone of source FNN)
    - Fixed structure (no growing/pruning)
    - Uses knowledge regularisation (fixed alpha=0.9, beta=0.1)
    - Weaker than DK-SOFNN: no structure adjustment, no adaptive alpha

  RS-SOFNN (Rule Sensitivity SOFNN):
    - Self-organizing: grows/prunes based on Sensitivity (M) index only
    - NO source knowledge transfer
    - No parameter regularisation from source
    - Weaker: ignores R and C, no domain knowledge

  APSO-FNN (Adaptive PSO-FNN):
    - Self-organizing using all three indices (R, M, C)
    - NO source knowledge transfer
    - Data-driven only (alpha=1, beta=0)
    - Weaker: ignores source knowledge entirely

  Fixed-Structure FNN (used for Figs 13-15):
    - Fixed K rules (K=11, 13, or 15), no structure adjustment
    - Used to show DK-SOFNN's adaptive structure advantage

--------------------------------------------------------------------------------
CELL 8 — Evaluation Metrics
--------------------------------------------------------------------------------
PURPOSE : Define the three metrics used for performance comparison.

  RMSE  (Root Mean Square Error, Eq. 42):
    sqrt( mean( (y_pred - y_true)^2 ) )
    → Lower is better. Penalises large errors heavily.

  sMAPE (Symmetric MAPE, Eq. 43):
    mean( 2*|y_pred - y_true| / (|y_pred| + |y_true|) )
    → Lower is better. Scale-independent percentage error.

  MASE  (Mean Absolute Scaled Error, Eq. 44):
    mean(|y_pred - y_true|) / mean(|y_true[t] - y_true[t-1]|)
    → Lower is better. Compares to naive 1-step forecast.

--------------------------------------------------------------------------------
CELL 10 — 30-Run Experiment (Main Comparison Table)
--------------------------------------------------------------------------------
PURPOSE : Repeat the experiment 30 times with different random seeds to get
          statistically reliable results (mean ± std).

WHY 30 RUNS?
  Neural network training is stochastic (random initialization). A single run
  may be lucky or unlucky. 30 runs give a reliable estimate of true performance.

WHAT EACH RUN DOES (seed = run * 37):
  1. Train source FNN (K0=20, 300 epochs)
  2. Train DK-SOFNN target (300 epochs)
  3. Train GM-FTL (300 epochs)
  4. Train RS-SOFNN (300 epochs)
  5. Train APSO-FNN (300 epochs)
  6. Evaluate all on clean test labels → RMSE, sMAPE, MASE
  7. Save best run (lowest DK-SOFNN RMSE) for plotting

LIVE STATUS:
  After each run, a status line is printed showing RMSE for each method,
  elapsed time, and estimated time remaining (ETA).
